# claude-para-tcc
## Introdução: caso específico — trabalho acadêmico
O maior custo de tokens no caso de tarefas envolvendo TCC acadêmico não é a skill — são as seções do trabalho escrito que precisam estar no contexto para o Claude manter coerência com o restante do trabalho. Isso é inevitável.

## Conclusão prática
Skills não são uma ferramenta de otimização de tokens — são uma ferramenta de qualidade e reaproveitamento de instruções.
Para esse tipo de trabalho acadêmico, o mais eficiente seria:
1º) Criar uma skill com as convenções do trabalho (norma, estrutura, tom, área do conhecimento);
2º) Enviar apenas as seções relevantes para o contexto imediato (não o trabalho inteiro se não for necessário);
3º) Pedir as seções faltantes de forma incremental.
Assim há equilíbrio de qualidade + custo de token.
